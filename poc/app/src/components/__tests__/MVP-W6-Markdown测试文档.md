# MVP W6 Markdown 渲染 — 测试文档

> **阶段**：W6 Markdown 渲染  
> **目标**：验证 Markdown/代码高亮/消息渲染/主题切换  
> **测试框架**：Jest + TypeScript

---

## 一、MarkdownRenderer 测试

### 1.1 基础渲染

| 用例 | 输入 | 预期输出 |
|------|------|---------|
| 纯文本 | `Hello World` | 显示文本 |
| 标题 | `# 标题` | 渲染为标题样式 |
| 列表 | `- item1\n- item2` | 渲染为列表 |
| 链接 | `[text](url)` | 渲染为可点击链接 |
| 图片 | `![alt](url)` | 渲染为图片 |
| 粗体 | `**bold**` | 渲染为粗体 |
| 斜体 | `*italic*` | 渲染为斜体 |
| 代码块 | `` `code` `` | 渲染为行内代码 |
| 引用 | `> quote` | 渲染为引用样式 |

### 1.2 GFM 扩展

| 用例 | 输入 | 预期输出 |
|------|------|---------|
| 表格 | `\| a \| b \|` | 渲染为表格 |
| 任务列表 | `- [ ] task` | 渲染为复选框 |
| 删除线 | `~~text~~` | 渲染为删除线 |

### 1.3 边界情况

| 用例 | 输入 | 预期输出 |
|------|------|---------|
| 空字符串 | `''` | 渲染为空 |
| 超长文本 | 10000 字符 | 不崩溃，可滚动 |
| 特殊字符 | `<>&"'` | 正确转义，不 XSS |

## 三、MessageRenderer 测试

### 3.1 消息类型

| 用例 | 输入 | 预期输出 |
|------|------|---------|
| 用户消息 | role: 'user' | 右对齐气泡 |
| AI 消息 | role: 'assistant' | 左对齐无气泡 |
| 系统消息 | role: 'system' | 居中显示 |

### 3.2 混合内容

| 用例 | 输入 | 预期输出 |
|------|------|---------|
| Markdown + 代码 | `# Title\n\`\`\`js\ncode\`\`\`` | 正确渲染 |
| Markdown + 链接 | `[link](url)` | 可点击链接 |
| 长消息 | 1000 字符 | 可滚动 |

## 四、设计 Token 测试

### 4.1 亮色模式

| 用例 | 输入 | 预期输出 |
|------|------|---------|
| 默认主题 | mode: 'light' | 使用 lightTokens |
| 背景色 | paper | `#faf6ee` |
| 文字色 | ink | `#1c1612` |
| 强调色 | accent-warm | `#c26d3a` |

### 4.2 暗色模式

| 用例 | 输入 | 预期输出 |
|------|------|---------|
| 暗色主题 | mode: 'dark' | 使用 darkTokens |
| 背景色 | paper | `#1a1614` |
| 文字色 | ink | `#e4dcd4` |
| 强调色 | accent-warm | `#d4803f` |

### 4.3 主题切换

| 用例 | 操作 | 预期结果 |
|------|------|---------|
| 切换到暗色 | setMode('dark') | 立即生效 |
| 切换到亮色 | setMode('light') | 立即生效 |
| 跟随系统 | setMode('system') | 跟随系统设置 |
