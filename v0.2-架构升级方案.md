# MyAgents Companion v0.2 架构升级方案

> **文档版本**：v1.0  
> **创建日期**：2026-06-14  
> **前置条件**：v0.1 PoC 已完成（Phase 0 + Phase 1 MVP 部分）  
> **目标**：解决 PoC 阶段的架构债务，建立可维护、可扩展的生产级架构

## 一、背景与动机

### 1.1 PoC 阶段架构问题汇总

| # | 问题 | 根因 | 影响 | 严重度 |

## 二、架构升级方案

### 2.1 问题 1：滚动位置丢失

**根因**：无导航库，每次切换页面组件完全卸载，ScrollView 状态丢失。

**方案**：引入 React Navigation

```typescript
// src/navigation/AppNavigator.tsx
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';

const Stack = createNativeStackNavigator();

export function AppNavigator() {
  return (
    <NavigationContainer>
      <Stack.Navigator>
        <Stack.Screen name="Home" component={HomeScreen} />
        <Stack.Screen name="Connection" component={ConnectionScreen} />
        <Stack.Screen name="SessionList" component={SessionListScreen} />
        <Stack.Screen name="Chat" component={ChatScreen} />
        <Stack.Screen name="Helper" component={HelperScreen} />
      </Stack.Navigator>
    </NavigationContainer>
  );
}
```

**收益**：
- 页面栈自动保持，返回时恢复滚动位置
- 深度链接原生支持
- 转场动画流畅
- 状态栏/标题栏自动管理

**依赖**：
```json
{
  "@react-navigation/native": "^7.0.0",
  "@react-navigation/native-stack": "^7.0.0",
  "react-native-screens": "^4.0.0",
```

### 2.2 问题 2：消息格式混乱

**根因**：无统一数据层，API 返回格式、Store 存储格式、UI 渲染格式各自为政。

**方案**：定义统一的类型系统 + API Service 层

```typescript
// src/types/message.ts
export interface Message {
  id: string;
  sessionId: string;
  role: 'user' | 'assistant' | 'system';
  content: string;           // 纯文本内容
  rawContent?: any;          // 原始 ContentBlock[]（用于调试）
  toolCalls?: ToolCall[];    // 工具调用列表
  thinking?: string;         // 思考过程
  createdAt: number;
  status: 'sending' | 'sent' | 'error';
}

export interface ToolCall {
  id: string;
  name: string;
  input: string;
  output?: string;
  status: 'running' | 'completed' | 'error';
  startTime?: number;
  endTime?: number;
}

// src/types/session.ts
export interface Session {
  id: string;
  title: string;
  lastMessageAt: number;
  messageCount: number;
  isInternal: boolean;
  agentDir?: string;
  source?: string;
}

// src/types/connection.ts
export interface ConnectionConfig {
  host: string;
  port: number;
  token?: string;
  pairedAt?: number;
  lastConnectedAt?: number;
}
```

```typescript
// src/services/ApiService.ts
export class ApiService {
  private baseUrl: string;
  private token: string | null = null;

  constructor(config: ConnectionConfig) {
    this.baseUrl = `http://${config.host}:${config.port}`;
    this.token = config.token || null;
  }

  async getSessionList(): Promise<Session[]> {
    const data = await this.get<{ sessions: any[] }>('/api/session/list');
    return data.sessions.map(this.normalizeSession);
  }

  async getMessages(sessionId: string): Promise<Message[]> {
    const data = await this.get<{ messages: any[] }>(
      `/api/session/messages?sessionId=${sessionId}`
    );
    return data.messages.map(this.normalizeMessage);
  }

  // 统一的格式转换
  private normalizeSession(raw: any): Session {
    return {
      id: raw.id,
      title: raw.title || '未命名会话',
      lastMessageAt: raw.lastMessageAt || Date.now(),
      messageCount: raw.messageCount || 0,
      isInternal: !!raw.internal,
      agentDir: raw.agentDir,
      source: raw.source,
    };
  }

  private normalizeMessage(raw: any): Message {
    return {
      id: raw.id || raw.uuid,
      sessionId: raw.sessionId,
      role: raw.role,
      content: this.extractText(raw.content),
      rawContent: raw.rawContent,
      toolCalls: this.extractToolCalls(raw.content),
      thinking: this.extractThinking(raw.content),
      createdAt: raw.createdAt || Date.now(),
      status: 'sent',
    };
  }
}
```

**收益**：
- 前端只操作统一的 Message/Session 类型
- 格式转换逻辑集中，不散落各处
- 后端格式变化只需改 ApiService
- TypeScript 类型检查覆盖全流程

### 2.3 问题 3：会话不同步

**根因**：无全局状态管理，会话状态散落在各组件的 useState 中。

**方案**：Zustand 全局状态

```typescript
// src/store/sessionStore.ts
import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';
import AsyncStorage from '@react-native-async-storage/async-storage';

interface SessionState {
  sessions: Session[];
  currentSessionId: string | null;
  loading: boolean;
  error: string | null;
}

interface SessionActions {
  loadSessions: () => Promise<void>;
  selectSession: (sessionId: string) => void;
  appendMessage: (sessionId: string, message: Message) => void;
  updateSession: (sessionId: string, updates: Partial<Session>) => void;
  clearError: () => void;
}

export const useSessionStore = create<SessionState & SessionActions>()(
  persist(
    (set, get) => ({
      sessions: [],
      currentSessionId: null,
      loading: false,
      error: null,

      loadSessions: async () => {
        set({ loading: true, error: null });
        try {
          const api = useConnectionStore.getState().getApi();
          const sessions = await api.getSessionList();
          set({ sessions, loading: false });
        } catch (err) {
          set({ error: err.message, loading: false });
        }
      },

      selectSession: (sessionId) => {
        set({ currentSessionId: sessionId });
        // 自动加载消息
        useMessageStore.getState().loadMessages(sessionId);
      },

      appendMessage: (sessionId, message) => {
        set((state) => ({
          sessions: state.sessions.map((s) =>
            s.id === sessionId
              ? { ...s, messageCount: s.messageCount + 1, lastMessageAt: Date.now() }
              : s
          ),
        }));
      },

      updateSession: (sessionId, updates) => {
        set((state) => ({
          sessions: state.sessions.map((s) =>
            s.id === sessionId ? { ...s, ...updates } : s
          ),
        }));
      },

      clearError: () => set({ error: null }),
    }),
    {
      name: 'session-storage',
      storage: createJSONStorage(() => AsyncStorage),
      partialize: (state) => ({
        currentSessionId: state.currentSessionId,
      }),
    }
  )
);
```

**收益**：
- 会话状态全局一致，任何组件都能访问
- 切换会话时自动加载消息
- 持久化当前会话 ID，重启后恢复
- 类型安全，编译时检查

### 2.5 问题 5：库兼容崩溃

**根因**：无错误边界，第三方库异常直接导致白屏。

**方案**：全局 Error Boundary

```typescript
// src/components/ErrorBoundary.tsx
import React, { Component, ErrorInfo, ReactNode } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';

interface Props {
  children: ReactNode;
  fallback?: ReactNode;
}

interface State {
  hasError: boolean;
  error: Error | null;
}

export class ErrorBoundary extends Component<Props, State> {
  state: State = { hasError: false, error: null };

  static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error };
  }

  componentDidCatch(error: Error, errorInfo: ErrorInfo) {
    // 上报错误到日志服务
    console.error('[ErrorBoundary]', error, errorInfo);
  }

  handleRetry = () => {
    this.setState({ hasError: false, error: null });
  };

  render() {
    if (this.state.hasError) {
      if (this.props.fallback) {
        return this.props.fallback;
      }

      return (
        <View style={styles.container}>
          <Text style={styles.title}>渲染出错</Text>
          <Text style={styles.message}>{this.state.error?.message}</Text>
          <TouchableOpacity style={styles.button} onPress={this.handleRetry}>
            <Text style={styles.buttonText}>重试</Text>
          </TouchableOpacity>
        </View>
      );
    }

    return this.props.children;
  }
}

// src/App.tsx
export default function App() {
  return (
    <ErrorBoundary>
      <NavigationContainer>
        {/* ... */}
      </NavigationContainer>
    </ErrorBoundary>
  );
}
```

**收益**：
- 第三方库异常不会导致白屏
- 用户看到友好的错误提示
- 支持重试恢复
- 错误可追踪

### 2.6 问题 6：本地存储丢失

**根因**：无持久化，App 重启后所有数据丢失。

**方案**：AsyncStorage + MMKV

```typescript
// src/services/StorageService.ts
import AsyncStorage from '@react-native-async-storage/async-storage';
import { MMKV } from 'react-native-mmkv';

// MMKV 用于高频读写（性能更好）
const mmkv = new MMKV();

// AsyncStorage 用于大容量存储（兼容性好）
export class StorageService {
  // 连接配置（MMKV）
  static saveConnection(config: ConnectionConfig): void {
    mmkv.set('connection', JSON.stringify(config));
  }

  static getConnection(): ConnectionConfig | null {
    const data = mmkv.getString('connection');
    return data ? JSON.parse(data) : null;
  }

  // 会话缓存（AsyncStorage）
  static async saveSessionCache(sessionId: string, messages: Message[]): Promise<void> {
    const key = `session_${sessionId}`;
    await AsyncStorage.setItem(key, JSON.stringify(messages));
  }

  static async getSessionCache(sessionId: string): Promise<Message[]> {
    const key = `session_${sessionId}`;
    const data = await AsyncStorage.getItem(key);
    return data ? JSON.parse(data) : [];
  }

  // 清理过期缓存
  static async clearExpiredCache(maxAgeDays: number = 30): Promise<void> {
    const keys = await AsyncStorage.getAllKeys();
    const now = Date.now();
    const maxAge = maxAgeDays * 24 * 60 * 60 * 1000;

    for (const key of keys) {
      if (key.startsWith('session_')) {
        const data = await AsyncStorage.getItem(key);
        if (data) {
          const messages = JSON.parse(data);
          const lastMessage = messages[messages.length - 1];
          if (lastMessage && now - lastMessage.createdAt > maxAge) {
            await AsyncStorage.removeItem(key);
          }
        }
      }
    }
  }
}
```

**收益**：
- 连接配置持久化，重启后自动恢复
- 会话消息本地缓存，离线可查看
- 自动清理过期数据
- MMKV 高性能，AsyncStorage 大容量

### 2.7 问题 7：会话过滤误伤

**根因**：硬编码过滤逻辑，误伤正常会话。

**方案**：基于 agentDir 精确匹配

```typescript
// src/utils/sessionFilter.ts
import { homedir } from 'os';

// 需要过滤的会话类型
const FILTERED_PATTERNS = [
  // 小助理会话
  {
    check: (s: Session) => {
      const homeDir = homedir().replace(/\\/g, '/');
      const agentDir = s.agentDir?.replace(/\\/g, '/');
      return agentDir === `${homeDir}/.myagents`;
    },
    reason: '小助理会话',
  },
  // Cron 会话
  {
    check: (s: Session) => !!s.cronTaskId,
    reason: '定时任务会话',
  },
  // 内部会话
  {
    check: (s: Session) => !!s.isInternal,
    reason: '内部会话',
  },
];

export function filterUserSessions(sessions: Session[]): Session[] {
  return sessions.filter((session) => {
    for (const pattern of FILTERED_PATTERNS) {
      if (pattern.check(session)) {
        return false;
      }
    }
    return true;
  });
}

// 调试用：记录被过滤的会话
export function getFilteredSessions(sessions: Session[]): Array<{ session: Session; reason: string }> {
  const filtered: Array<{ session: Session; reason: string }> = [];

  for (const session of sessions) {
    for (const pattern of FILTERED_PATTERNS) {
      if (pattern.check(session)) {
        filtered.push({ session, reason: pattern.reason });
        break;
      }
    }
  }

  return filtered;
}
```

**收益**：
- 过滤逻辑可配置，不硬编码
- 精确匹配 agentDir，不误伤
- 支持调试，可查看被过滤的会话
- 易于扩展新的过滤规则
