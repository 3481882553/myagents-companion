## W11：内置小助理 MVP（5 个工作日）

> **核心目标**：手机端小助理 = 连接客户端诊断 + 手机端自诊断 + 提交 Issue
> **设计文档**：`v3.0-详细设计.md` 第十章「内置小助理详细设计」。

### 目标
实现小助理的移动端版本，两个核心能力：
1. **连接客户端** — 诊断桌面端问题（读日志、查状态、执行 skill）
2. **手机端自诊断** — 查日志、检查连接、生成诊断报告、提交 GitHub Issue

### 前置条件
- ✅ W8-9 Sidecar 对接完成（消息加载/发送可用）
- ✅ W10 消息发送可用
- ⏳ W6+ 消息渲染升级（工具调用可视化）
- ⏳ SSE 流式接收（实时看到 AI 回复）

### 架构设计

```
手机端小助理
    │
    ├── 会话入口（首页卡片 / 专属页面）
    │
    ├── 消息收发（复用 W10 的 /api/session/send + SSE）
    │
    ├── Admin API 代理（通过 Sidecar 查询状态）
    │   ├── GET /api/config/provider   → Provider 状态
    │   ├── GET /api/config/mcp        → MCP 服务器状态
    │   ├── GET /api/admin/cron/list   → 定时任务列表
    │   └── GET /api/health            → 系统健康状态
    │
    └── 日志查询（通过 Sidecar 搜索日志）
        └── GET /api/admin/logs?keyword=xxx
```

### 与桌面端小助理的能力对比

| 能力 | 桌面端 | 手机端 |
|------|--------|--------|
| 对话问答 | ✅ Chat | ✅ Chat（复用） |
| `/support` 诊断 | ✅ Skill | ✅ Skill（复用） |
| `/myagents-cli` 管理 | ✅ Skill | ✅ Skill（复用） |
| 读日志 | ✅ Bash | ✅ Bash（复用） |
| 配置 Provider | ✅ Settings UI | ✅ Admin API 代理 |
| 实时流式回复 | ✅ SSE | ⏳ W10 SSE |
| 会话隔离 | ✅ project.internal 过滤 | ⏳ 需实现 |

> **会话隔离**：桌面端通过 `project.internal === true` 过滤小助理会话，手机端需要在 SessionListScreen 中过滤 `agentDir` 包含 `.myagents` 的会话。

> **关键设计决策**：手机端小助理**不需要重新实现**任何功能——它只是一个新的 UI 入口，所有能力都通过 Sidecar 后端提供。桌面端的 `CLAUDE.md`、`support` skill、`myagents-cli` skill 在手机端同样可用。
