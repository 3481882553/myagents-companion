## 三、v0.4 核心任务

### W1：SSE 架构修复（5 天）

**目标**：解决 send 和 SSE 走不同 Sidecar 的问题

**方案**：让 mobile-server 的所有请求（send + stream）都转发到同一个 Sidecar。

```
当前架构：
  Phone → mobile-server(32103) → send → 源码 Sidecar(31500)
  Phone → mobile-server(32103) → SSE  → 安装版 Sidecar(31430)  ← 不同进程！

目标架构：
  Phone → mobile-server(32103) → send → 安装版 Sidecar(port in sidecar.port)
  Phone → mobile-server(32103) → SSE  → 安装版 Sidecar(port in sidecar.port)  ← 同一个进程！
```

**改动**：
1. `mobile-server.ts`：`handleSessionSend` 也转发到安装版 Sidecar 端口（而非源码 Sidecar 端口）
2. 源码 Sidecar 不需要自己的 session，所有请求代理到安装版
3. 修复 SseClient 重试计数 bug

**验证标准**：
- 从手机发消息 → 安装版 Sidecar 处理 → AI 回复 → SSE 推送回手机
- 手机上能看到实时流式回复

### W2：Widget 渲染（5 天）

**前置条件**：W1 SSE 架构修复完成

**设计文档**：`v3.0-详细设计.md` §8

**实现**：
1. `WidgetRenderer.tsx`（WebView 沙箱）
2. `sanitize` 安全过滤
3. Widget 通信协议（postMessage: ready/resize/link/error/theme）
4. MessageRenderer 集成（识别 `widget` 类型 ContentBlock）
5. 降级策略（渲染失败 → 代码文本）

**验证标准**：
- 发送含 `<generative-ui-widget>` 的消息 → Widget 正确渲染
- Widget 内链接点击 → 系统浏览器打开
- 暗色主题切换 → Widget 同步

### W3：SQLite 离线缓存 + 测试（5 天）

**设计文档**：`v3.0-概要设计.md` §8

**实现**：
1. SQLite 表结构 + 迁移
2. 缓存读写 + LRU 淘汰
3. ChatScreen 离线模式（断网查看历史）
4. 全量回归测试

**验证标准**：
- 断网后查看历史消息
- 联网后自动同步
- 缓存大小不超过 50MB
