# MyAgents Android 端 Phase 1 需求文档

## 项目代号：MyAgents Mobile Shell

> 目标：为 MyAgents 构建一个 Android 原生渲染壳，解决 IM 接入导致的可视化丢失问题。逻辑层完全复用远程 Sidecar，本地仅负责渲染与交互。

---

## 1. 项目定位

| 维度 | 定义 |
|------|------|
| **不是什么** | 不是 MyAgents 的"移植版"，不复制桌面端 Sidecar、终端、浏览器等逻辑 |
| **是什么** | 一个**渲染与交互终端**，通过 WebSocket 连接云端/自托管 Sidecar，将 MyAgents 的可视化能力完整带到移动端 |
| **核心差异** | 桌面端 = Sidecar + 前端；移动端 = 远程 Sidecar + 本地渲染壳 |

## 3. 非功能需求

### 3.1 性能

| 需求 ID | 指标 | 目标 |
|---------|------|------|
| N1.1 | 首屏加载时间 | < 2s（从点击图标到显示会话列表） |
| N1.2 | 消息渲染延迟 | < 100ms（单条消息从接收到完整渲染） |
| N1.3 | 流式渲染帧率 | > 30fps（打字机效果不卡顿） |
| N1.4 | 内存占用 | < 150MB（正常使用状态） |
| N1.5 | APK 体积 | < 10MB（不含可选模块） |

### 3.2 兼容性

| 需求 ID | 指标 | 目标 |
|---------|------|------|
| N2.1 | Android 版本 | API 26+（Android 8.0） |
| N2.2 | 屏幕适配 | 手机 + 平板，支持分屏 |
| N2.3 | 网络环境 | 适配弱网、4G/5G/WiFi 切换 |

### 3.3 安全

| 需求 ID | 指标 | 目标 |
|---------|------|------|
| N3.1 | 数据传输 | WebSocket over TLS（WSS） |
| N3.2 | 本地存储 | 敏感数据（API Key）加密存储（EncryptedSharedPreferences） |
| N3.3 | 证书校验 | 不支持自签名证书（除非用户手动导入） |

## 4. 技术栈

| 层级 | 技术选型 | 理由 |
|------|---------|------|
| **UI 框架** | Jetpack Compose | 现代 Android UI，声明式，与 Markdown 渲染组件天然契合 |
| **网络层** | Ktor Client (WebSocket) | Kotlin 原生，协程友好，支持 SSE 流式 |
| **Markdown 解析** | Jetpack Markdown / 自研解析器 | 基础解析用现成库，扩展语法（Mermaid/LaTeX）自研渲染 |
| **LaTeX 渲染** | KaTeX Android 移植 | 体积小(~500KB)，速度快，质量高 |
| **Mermaid 渲染** | WebView 桥接 Mermaid.js | 复用成熟生态，按需加载 |
| **代码高亮** | Prism.js (WebView) 或 Treesitter | 语言覆盖广，主题可定制 |
| **本地存储** | Room (SQLite) | 会话列表、消息缓存、用户配置 |
| **依赖注入** | Hilt | Google 官方，与 Compose 集成好 |
| **图片加载** | Coil | Kotlin 友好，支持 Compose，支持 SVG/GIF |

## 6. 与桌面端的差异对照

| 功能 | 桌面端 (MyAgents) | Android 端 (Mobile Shell) |
|------|------------------|---------------------------|
| Sidecar | 本地 Node.js 进程 | 远程 WebSocket 连接 |
| 终端 | `portable-pty` 原生 PTY | ❌ 不支持（Phase 1 砍掉） |
| 内嵌浏览器 | Tauri Multi-Webview | ❌ 不支持（Phase 1 砍掉） |
| 文件系统 | 自由访问 | SAF 受限访问（仅上传/下载） |
| 全局快捷键 | 系统级注册 | ❌ 不支持 |
| 系统托盘 | 常驻后台 | 前台 Service（可选） |
| MCP/工具执行 | 本地 spawn 进程 | 远程 Sidecar 执行，结果返回 |
| 会话隔离 | 进程级（每 Tab 独立 Sidecar） | 连接级（WebSocket 多路复用） |

## 8. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| WebSocket 在弱网下不稳定 | 高 | 实现指数退避重连，消息队列缓冲，断网提示 |
| Mermaid/LaTeX 渲染性能差 | 中 | WebView 懒加载，预渲染缓存，大图降级 |
| Sidecar API 不兼容 | 高 | Phase 0 先定义 WebSocket 协议契约，与桌面端同步 |
| APK 体积超标 | 中 | Mermaid.js 按需下载，KaTeX 精简字体子集 |
| 用户不接受"远程 Sidecar" | 高 | 提供自托管文档，局域网模式，强调数据隐私可控 |

## 9. 附录：WebSocket 协议草案（与 Sidecar 的接口）

### 9.1 连接建立

```json
// Client → Server
{
  "type": "auth",
  "token": "jwt_token_here",
  "client": "myagents-android",
  "version": "1.0.0"
}

// Server → Client
{
  "type": "auth_result",
  "success": true,
  "session_id": "uuid"
}
```

### 9.2 发送消息

```json
// Client → Server
{
  "type": "chat",
  "session_id": "uuid",
  "message": {
    "role": "user",
    "content": "你好",
    "attachments": []
  }
}
```

### 9.3 接收流式响应

```json
// Server → Client (流式，多条)
{
  "type": "delta",
  "session_id": "uuid",
  "delta": {
    "content": "你",
    "finish_reason": null
  }
}

// 最后一条
{
  "type": "delta",
  "session_id": "uuid",
  "delta": {
    "content": "",
    "finish_reason": "stop"
  }
}
```

### 9.4 工具调用结果

```json
// Server → Client
{
  "type": "tool_call",
  "session_id": "uuid",
  "tool_call": {
    "id": "call_xxx",
    "name": "read_file",
    "arguments": {"path": "/tmp/test.txt"}
  }
}

// Client → Server（如果需要用户确认）
{
  "type": "tool_result",
  "tool_call_id": "call_xxx",
  "result": {
    "success": true,
    "output": "file content here"
  }
}
```
