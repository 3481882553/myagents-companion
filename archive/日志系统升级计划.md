### 1.2 桌面端日志方案

桌面端（`D:\MyAgents-source`）的日志系统：

```
~/.myagents/logs/
  └── {date}-{sessionId}.log
      ├── [REACT] 前端 UI 日志
      ├── [RUST] Tauri/Proxy/Sidecar 生命周期
      └── [NODE] Sidecar/Provider/MCP/SDK 日志

统一输出：console.log → monkey-patch → 写入文件
启动自检行：[boot] pid=... port=... node=... provider=... mcp=... model=...
```

关键设计：
- **统一日志目录**：所有来源的日志写入 `~/.myagents/logs/`
- **启动自检行**：`[boot]` 行包含完整环境信息，一行定位启动问题
- **Session 级命名**：`{date}-{sessionId}.log`
- **日志保留策略**：`log-retention.ts` 管理大小和时间

### 1.3 用户实际遇到的问题

| 问题 | 需要什么日志才能看到 | 当前有吗 |
|------|-------------------|---------|
| 连接 Aborted | fetch 的具体错误（DNS/超时/拒绝） | ❌ |
| 发送消息无响应 | API 请求/响应的 status code | ❌ |
| SSE 断开 | SSE 连接状态变化、心跳超时 | ❌ |
| Token 过期 | JWT 验证失败的具体原因 | ❌ |
| 存储异常 | AsyncStorage/MMKV 操作成功/失败 | ❌ |
| 渲染异常 | 消息渲染前后的数据内容 | ❌ |

### 1.4 adb logcat 过滤能力

```bash
# 按 TAG 过滤（最常用）
adb logcat -s ReactNativeJS:I

# 按多个 TAG
adb logcat -s ReactNativeJS:I MobileLogger:I

# 按关键字
adb logcat | grep -i "ConnectionScreen\|SseClient\|error"

# 时间窗口
adb logcat -d -t 200
```

**RN 在 Android 上的日志输出**：`ReactNativeJS` TAG，级别为 `I`（info）或 `E`（error）。

## 二、升级方案

### 2.1 设计原则

| 原则 | 说明 |
|------|------|
| **最小侵入** | 不改现有 API，只在关键路径加日志 |
| **结构化** | TAG 统一格式 `[Module] message`，便于 grep |
| **按需关闭** | `__DEV__` 下输出，Release 下可选关闭 |
| **不依赖外部库** | 只用 `console.log`，由 MobileLogger 拦截 |

### 2.2 统一 TAG 格式

```
[Module] [时间戳] 消息

示例：
[SseClient] 连接 http://localhost:32102/chat/stream
[SseClient] 状态变化: DISCONNECTED → CONNECTING
[SseClient] 状态变化: CONNECTING → CONNECTED
[ApiService] GET /api/session/list 120ms 200
[ApiService] POST /api/session/send 45ms 200
[ConnectionManager] fetch /health/live 超时 5000ms
[ConnectionManager] 错误: ECONNREFUSED 127.0.0.1:32102
[connectionStore] connect: host=10.0.2.2 port=32102
[messageStore] appendDelta: sessionId=s1 msgId=m1 len=45
[SseEvent] message-chunk: text="Hello World" len=11
```

### 2.3 日志等级

| 等级 | 输出条件 | 用途 |
|------|---------|------|
| `debug` | `__DEV__` only | 开发调试（函数入口/中间状态） |
| `info` | 始终输出 | 连接/断开/发送/接收等生命周期事件 |
| `warn` | 始终输出 | 可恢复的错误（重试、Token 即将过期） |
| `error` | 始终输出 | 不可恢复的错误（连接失败、API 错误） |

## 三、需要添加日志的关键点

### 3.1 ConnectionManager（优先级最高）

```
connect() 开始: 连接目标地址
  → health check 请求发送
  → health check 响应（status/duration）
  → 配对请求发送
  → 配对响应（token 长度）
  → 连接完成/失败（错误类型）
reconnect(): 重连触发（原因、延迟、次数）
disconnect(): 断开原因
```

### 3.2 ApiService

```
每个请求: method + url + duration + status
每个错误: method + url + error.name + error.message
```

### 3.3 SseClient

```
connect(): 目标 URL
state变化: DISCONNECTED → CONNECTING → CONNECTED
事件: 收到事件类型 + 大小
error: 错误类型（连接/解析/超时）
reconnect: 重连计数 + 延迟
```

### 3.4 messageStore

```
appendDelta: sessionId + messageId + 新增长度
finalizeMessage: sessionId + messageId + 最终长度
upsertToolBlock: toolId + name
loadMessages: sessionId + 消息数量
sendMessage: sessionId + 文本长度
```

### 3.5 connectionStore

```
connect: host + port
setConnected: token 前 4 位
setError: 错误类型 + 错误信息
```

### 3.6 StorageService

```
saveConnection: 成功/失败
loadConnection: 成功/失败 + 结果
saveToken: 成功/失败
```
