## 二、问题 1：代码结构分裂

### 现状

```
D:\myagents-android\
│
├── src/                          ← v0.2 新代码（孤立）
│   ├── App.tsx
│   ├── components/
│   ├── navigation/
│   ├── screens/
│   ├── services/
│   ├── store/
│   ├── types/
│   └── utils/
│
└── poc/app/                      ← RN 项目（真正运行）
    ├── package.json              ← 所有依赖在这里
    ├── node_modules/
    ├── metro.config.js
    ├── index.js                  ← 入口：import './App'
    ├── index.v2.js               ← 桥接：import '../../src/App'（未启用）
    └── src/                      ← PoC 旧代码（实际运行）
        ├── screens/
        ├── services/
        └── store/
```

### 根因

`v0.2-架构升级方案.md` 设计 `src/` 为代码目录、`poc/app/` 为运行时环境，通过 `index.v2.js` 桥接。但实施时：

1. **Metro 配置遗漏**：`metro.config.js` 缺少 `watchFolders`，Metro 默认不解析项目外文件
2. **node_modules 不可达**：`src/` 不在 `poc/app/` 内，无法回溯到 `node_modules`
3. **两套代码并存**：PoC 旧代码和 v0.2 新代码各有一套 screens/services/store

### 后果

- 编辑 `src/` 代码后需要手动重启 Metro 并折腾 `watchFolders` 配置
- `index.js` 指向 PoC 旧代码，加了日志看到的是 PoC 的堆栈
- 调试连接问题时 API 方法名不匹配（`getSessions` vs `getSessionList`）
- `adb reverse` 端口转发断开后 app 白屏，排查链路长

### 修复方案

```
D:\myagents-android\poc\app\
├── package.json
├── metro.config.js
├── index.js                      ← import './src/App'
└── src/                          ← 合并后的唯一代码目录
    ├── App.tsx                   ← v0.2 ErrorBoundary + React Navigation
    ├── components/
    ├── navigation/
    ├── screens/
    ├── services/
    ├── store/
    ├── types/
    └── utils/
```

操作：
1. `src/` 代码覆盖 `poc/app/src/`
2. 删除外部 `D:\myagents-android\src\`
3. 删除 `poc/app/index.v2.js`（不再需要桥接）
4. 删除 `metro.config.js` 中的 `watchFolders`

## 四、问题 3：Jest 环境配置冲突

### 冲突源

| 配置位置 | 配置内容 |
|----------|---------|
| `jest.config.js` → `moduleNameMapper` | `'^react-native$': '<rootDir>/src/__tests__/mocks/react-native.js'` — 含 `StyleSheet`、`View`、`Text` 等 |
| `src/__tests__/setup.ts` → `jest.mock('react-native', ...)` | 不含 `StyleSheet`、`View`、`Text` — 只有 `Platform`、`useColorScheme`、`AppState`、`Linking` |

Jest 的 mock 优先级：`setupFiles` 中的 `jest.mock()` 会覆盖 `moduleNameMapper` 的同名映射。

**结果**：任何导入了 `StyleSheet`、`View` 等常用组件的测试文件，在 `require` 阶段直接崩 —— `Cannot read properties of undefined (reading 'create')`。

### 依赖缺失

`@testing-library/react-native@14.0.0` 内部 `require("test-renderer")`，但项目中安装的是 `react-test-renderer`。两个包名不同，导致 `@testing-library/react-native` 的 `render()` 无法加载。

### 修复方案

1. 删除 `setup.ts` 中重复的 `jest.mock('react-native', ...)`，只保留 `moduleNameMapper` 版本
2. 在 `jest.config.js` 的 `moduleNameMapper` 中添加 `'^test-renderer$': '<rootDir>/node_modules/react-test-renderer'`
3. 在 `jest.config.js` 的 `moduleNameMapper` 中添加 `'^@react-native/babel-preset'` 等缺失的模块映射
