## Context

v3.0 设计文档已完成并通过 SOLID 审查。下一步是 Phase 0 PoC 验证，目标是在 3 周内打通 RN → HTTP → Sidecar → SSE → UI 完整链路，验证关键技术风险。不写业务逻辑，只验证可行性。

**前置条件**：
- 设计文档 v3.0 已定稿（✅ 已完成）
- 桌面端 Sidecar 需手动改造支持 `0.0.0.0` 绑定（SC-01）
- 局域网环境：一台桌面端 + 一台 Android 手机

## W2：通信层 PoC（5 个工作日）

### 目标
验证移动端 SSE 客户端与桌面端 Sidecar 的通信能力，包括连接、认证、事件接收和重连。

### 任务清单

#### D1：通信验证（先 mock server，后改 Sidecar）

**Step 1：独立 mock server（无风险）**
- [ ] 用 Node.js + Hono 创建独立 mock server，模拟 Sidecar API
- [ ] 暴露 3 个端点：`POST /api/pair`（返回假 JWT）、`GET /api/session/list`（返回假数据）、`GET /api/session/stream`（返回假 SSE 事件）
- [ ] 绑定 `0.0.0.0:32101`，手机可访问
- [ ] 验证：手机 HTTP 请求 mock server → 收到响应
- [ ] 验证：手机 SSE 连接 mock server → 收到事件流

**Step 2：确认可行后改 Sidecar**
- [ ] 修改 `src/server/index.ts` 第 1816 行：`hostname` 从 `'127.0.0.1'` 改为可配置
- [ ] 新增环境变量 `MYAGENTS_MOBILE_PORT` 支持第二个 HTTP 服务
- [ ] 最小 JWT 认证中间件（硬编码 secret，PoC 用）
- [ ] 暴露 3 个端点：`/api/pair`、`/api/session/list`、`/api/session/stream`
- [ ] 验证：从另一台设备访问 `http://<desktop-ip>:<mobile-port>/api/session/list`

**交付物**：mock server 验证通过 + Sidecar 改造可被局域网内手机访问

**⚠️ 注意**：mock server 用于验证通信协议可行性，Sidecar 改造用于验证真实集成。两者都仅用于 PoC，不合并到主线。

#### D2：SSE 客户端实现
- [ ] 安装 `react-native-sse`
- [ ] 实现 `SseClient` 类（参考详细设计 §3.1 状态机）
- [ ] 实现 5 种状态：DISCONNECTED → CONNECTING → CONNECTED → RECONNECTING → RETRYING
- [ ] 实现事件解析：JSON 事件（`JSON_EVENTS` 集合中的 49 个）+ STRING 事件（`chat:message-chunk` 等）
- [ ] 实现优先级分发：critical（43 个）立即处理、coalescible（7 个）40ms 合并、droppable（4 个）忽略
- [ ] 验证：连接 Sidecar，接收 `chat:system-init` 事件

**交付物**：SSE 客户端可连接 Sidecar 并接收事件

#### D3：事件合并 + 心跳
- [ ] 实现 `EventCoalescer`（参考详细设计 §3.4）
- [ ] 40ms 时间窗口合并 coalescible 事件
- [ ] delta 类型拼接策略：`chat:message-chunk`、`chat:thinking-chunk` 等
- [ ] 心跳检测：15s 间隔，3 次未响应判定断开
- [ ] 验证：模拟 AI 回复，观察流式渲染效果

**交付物**：事件合并正常，流式渲染无闪烁

#### D4：认证流程 PoC
- [ ] 实现 `/api/pair` 配对流程：POST 配对码 → 获取 JWT Token
- [ ] Token 存储：`react-native-keychain`（Android Keystore / iOS Keychain）
- [ ] 后续请求携带 `Authorization: Bearer <token>`
- [ ] 验证：配对成功 → SSE 连接 → 接收事件

**交付物**：完整配对 → 认证 → 通信链路

#### D5：重连 + 网络切换测试
- [ ] 指数退避重连：初始 500ms，最大 30s，倍数 2x
- [ ] WiFi → 4G 切换测试：`ConnectivityManager.registerNetworkCallback` 监听
- [ ] 飞行模式开 → 关测试
- [ ] App 后台 30s → 回前台测试
- [ ] 记录重连成功率和延迟
- [ ] 输出 W2 验证报告

**交付物**：重连机制验证通过 + 验证报告

## Sidecar 改造方案（W2 D1 的详细说明）

### 改动范围

| 文件 | 改动 | 行数 |
|------|------|------|
| `src/server/index.ts` | 新增移动端 HTTP 服务 | ~30 行 |
| `src/server/mobile-auth.ts` | 最小 JWT 中间件 | ~50 行 |
| `src/server/mobile-routes.ts` | 3 个端点 | ~80 行 |
| `src-tauri/src/sidecar.rs` | 传入 `--mobile-port` | ~5 行 |

### 最小可用端点

```typescript
// /api/pair — 配对码验证，返回 JWT
// /api/session/list — 会话列表
// /api/session/stream — SSE 流（复用现有 broadcast）
```

### PoC 级别简化

- JWT secret 硬编码（PoC 用，不合并）
- 不实现 Token 续期（7 天有效，PoC 够用）
- 不实现 `/api/session/send`（W3 再加）
- 不实现 `/api/permission/respond`（W3 再加）

## 风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| `react-native-sse` 不兼容 Sidecar 协议 | 中 | W2 阻塞 | 备选：手动实现 EventSource polyfill |
| Mermaid WebView 渲染性能不达标 | 低 | W1 部分阻塞 | 降级为静态 SVG 方案 |
| Sidecar 改造影响桌面端稳定性 | 低 | W2 阻塞 | 独立进程/端口，不影响现有 127.0.0.1 |
| Android WebView 兼容性问题 | 中 | W1/W3 部分阻塞 | 测试多版本 WebView，必要时降级 |
