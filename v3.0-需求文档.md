> **项目代号**：MyAgents Mobile Companion  
> **文档版本**：v3.0  
> **更新日期**：2026-06-13  
> **文档性质**：需求文档（WHAT & WHY），不含实现方案

## 一、项目背景

### 1.1 现状问题

用户通过 IM 机器人（Telegram / 飞书 / 钉钉）在手机上使用 MyAgents，存在以下痛点：

| 痛点 | 具体表现 |
|------|---------|
| Markdown 渲染缺失 | 代码块、表格、LaTeX 公式在 IM 中显示为原始文本 |
| Mermaid 图表不可见 | 流程图、时序图在 IM 中变成代码文本 |
| 工具调用不可视 | Bash 输出、文件编辑等工具结果在 IM 中丢失结构 |
| 消息片段化 | 长 AI 回复被 IM 平台截断或分段发送 |

### 1.2 产品定位

**MyAgents Mobile Companion 是一个「IM 消息渲染增强器 + 轻量交互入口」。**

核心价值：用户在 IM 里收到 AI 回复后，点击链接即可在手机上获得与桌面端一致的渲染体验，并能完成简单的交互操作（发消息、审批权限）。

```
IM 机器人（微信/钉钉/飞书）
  │  摘要 + 深度链接
  ▼
MyAgents Companion App
  │  ✓ 完整 Markdown / 代码高亮 / Mermaid 图表
  │  ✓ 工具调用可视化
  │  ✓ 轻量交互（发消息 / 审批）
  │  ✓ 离线查看历史消息
  │  ✓ Widget 渲染
  │  ✗ 不运行 Agent / 不执行工具 / 不运行 Sidecar
  │
  │  HTTP / SSE（局域网或内网穿透）
  ▼
桌面端 MyAgents（Sidecar）
```

### 1.3 用户画像

| 角色 | 场景 | 核心需求 |
|------|------|---------|
| **开发者** | 在手机上查看 AI 的代码审查结果 | 代码高亮、diff 渲染、Bash 输出 |
| **知识工作者** | 在手机上查看 AI 生成的报告/分析 | Markdown 排版、表格、Mermaid 图表 |
| **多设备用户** | 离开电脑后继续与 AI 对话 | 发消息、查看历史、权限审批 |

## 二、用户故事

### US-01 从 IM 跳转查看 AI 回复

> 作为 IM 用户，我希望点击 AI 回复中的链接，直接在手机上看到完整渲染的内容，以便不需要等回到电脑前。

**验收标准**：
- 点击 IM 中的深度链接 → 打开 Companion App → 自动连接 Sidecar → 加载对应会话
- Markdown 渲染包含：GFM 表格、任务列表、删除线、代码块（语法高亮 + 复制按钮）、KaTeX 公式、Mermaid 流程图
- 首屏加载时间 ≤ 3 秒（局域网环境）

### US-02 扫码连接桌面端

> 作为首次用户，我希望扫一下桌面端显示的二维码就能连接，以便不需要手动输入 IP 和端口。

**验收标准**：
- 桌面端显示二维码，手机扫码 → App 自动连接
- 也支持手动输入 IP + 端口 + 配对码
- 连接状态实时显示（已连接 / 连接中 / 断开 / 重连中）

### US-03 查看 AI 工具调用过程

> 作为用户，我希望看到 AI 执行工具（Bash、文件读写、搜索等）的过程和结果，以便了解 AI 做了什么。

**验收标准**：
- 工具调用有明确的开始/结束标识
- Bash 输出以终端风格渲染，支持折叠长输出
- 文件读取/编辑显示代码 diff 高亮
- 工具调用支持展开/收起

### US-04 实时接收流式回复

> 作为用户，我希望像在电脑上一样看到 AI 逐字回复的效果，以便知道 AI 正在工作中。

**验收标准**：
- 逐字/逐块渲染，打字机效果
- 思考过程有"思考中"动画
- 工具调用有进度指示

### US-05 发送简单消息给 AI

> 作为用户，我希望在手机上也能给 AI 发消息，以便在离开电脑时继续对话。

**验收标准**：
- 输入文本消息并发送
- 发送后通过 SSE 接收 AI 回复
- 消息长度限制 4000 字符
- 不支持文件上传、终端操作等桌面专属能力

### US-06 审批工具权限

> 作为用户，我希望在手机上审批 AI 的工具使用请求，以便 AI 不会因为等待审批而卡住。

**验收标准**：
- 收到权限审批请求时显示弹窗
- 支持「允许」/「拒绝」操作
- 审批结果即时生效

### US-07 离线查看历史消息

> 作为用户，我希望在没有网络时也能查看之前看过的消息，以便在地铁等无网环境下回顾。

**验收标准**：
- 最近 30 天或 1000 条消息可离线查看（取较小值）
- 离线时显示只读视图，不支持发消息
- 恢复网络后自动同步最新消息
- 离线缓存不包含大型附件

### US-08 Widget 渲染

> 作为用户，我希望在手机上看到 AI 生成的交互式 Widget（图表、仪表盘等），以便获得完整的可视化体验。

**验收标准**：
- Widget 在安全沙箱中渲染，不能访问 App 数据
- 支持基础交互（点击、滚动）
- Widget 渲染失败时降级为静态截图或代码文本

### US-09 推送通知

> 作为用户，我希望在离开 App 后也能收到 AI 回复完成的通知，以便及时回来查看。

**验收标准**：
- AI 回复完成时发送本地推送通知
- 权限审批请求时发送推送通知
- 点击通知跳转到对应会话
- 支持 FCM（Android）+ APNs（iOS）
- 用户可在设置中关闭通知

### US-10 管理多个桌面端连接

> 作为多设备用户，我希望在 App 中保存多个桌面端的连接配置，以便快速切换不同的电脑。

**验收标准**：
- 保存多个 Sidecar 连接（IP + 端口 + Token）
- 快速切换连接
- 每个连接独立的会话列表

### US-11 使用内置小助理排查问题

> 作为用户，我希望在手机上直接和内置小助理对话，让它帮我诊断连接问题、配置错误，以便不需要等回到电脑前处理。

**验收标准**：
- 首页有显眼的小助理入口（卡片或浮动按钮）
- 点击进入标准聊天会话，小助理具备应用诊断和管理能力
- 开发阶段小助理可用于移动端自身的调试（连接失败、渲染异常等）

## 三、功能需求

### 3.1 P0 — MVP（必须）

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|---------|
| FR-01 | 会话消息阅读器 | 加载并渲染指定会话的完整消息历史 | Markdown/代码/KaTeX/Mermaid 完整渲染 |
| FR-02 | IM 深度链接 | 从 IM 消息点击链接跳转到 App | 点击 → 打开 App → 连接 → 加载会话 |
| FR-03 | Sidecar 连接管理 | 二维码扫描 / 手动输入连接 | 首次配对成功，连接状态可见 |
| FR-04 | 工具调用可视化 | 渲染 Bash/文件/搜索等工具结果 | 18 个工具组件正确渲染，支持展开/收起 |
| FR-05 | 流式消息接收 | SSE 流式渲染 AI 回复 | 逐字渲染无卡顿，延迟 ≤ 150ms |
| FR-06 | 内置小助理 | 首页入口进入小助理会话，预置 CLAUDE.md + support skill，通过 Sidecar Admin API 管理应用 | 对话正常，能诊断连接/配置问题，开发阶段可辅助调试 |

### 3.2 P1 — 轻交互（第二版）

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|---------|
| FR-07 | 发送消息 | 在 App 中发送文本消息给 AI | 消息成功送达，SSE 接收回复 |
| FR-08 | 权限审批 | 在 App 中审批工具权限请求 | 允许/拒绝操作即时生效 |
| FR-09 | 会话切换 | 在不同会话间切换 | 会话列表 + 搜索 + 切换加载 |

### 3.3 P2 — 增强体验（第三版）

| 编号 | 功能 | 描述 | 验收标准 |
|------|------|------|---------|
| FR-10 | 离线缓存 | 本地缓存历史消息 | 离线可查看，恢复网络自动同步 |
| FR-11 | Widget 沙箱渲染 | 安全渲染 Widget HTML | 沙箱隔离，渲染失败降级 |
| FR-12 | 多桌面实例管理 | 保存/切换多个 Sidecar 连接 | 多连接独立管理，切换无延迟 |
| FR-13 | 暗色/亮色主题 | 跟随系统或手动切换 | 主题切换即时生效，与桌面端色温一致 |
| FR-14 | 推送通知 | AI 回复完成 / 权限审批时推送 | 通知到达即时，点击跳转对应会话 |
| FR-15 | 设计一致性 | 与桌面端视觉风格统一 | 见 §3.4 设计一致性要求 |

### 3.4 设计一致性要求

移动端必须与桌面端保持**同一产品家族的视觉识别**。用户从桌面端切换到手机时，应感到是同一个产品，而非两个独立 App。

**必须对齐的设计要素**：

| 要素 | 要求 | 验收标准 |
|------|------|---------|
| **色调体系** | 暖色纸质风格（Warm Paper / Warm Night） | 主色调与桌面端一致，不使用冷色系 |
| **主题色** | Accent Warm: #c26d3a（亮）/ #d4803f（暗） | 按钮、链接、高亮使用同一色值 |
| **暗色模式** | 暖夜色温（paper #1a1614，elevated #242018） | 非纯黑背景，保持暖色温 |
| **消息布局** | AI 消息无气泡直接流式排列，用户消息有气泡 | 与桌面端消息渲染模式一致 |
| **代码块** | oneDark 语法主题，暖色代码背景 | 代码高亮颜色与桌面端一致 |
| **工具徽章** | 色彩编码系统（Emerald/Amber/Violet 等 9 色） | 每种工具类别使用与桌面端相同的颜色 |
| **图标** | Lucide 图标库 | 不使用 Emoji 替代图标 |
| **圆角体系** | 6/10/14/20/24px 五级 | 组件圆角使用体系内值 |
| **字体** | 系统字体栈（SF Pro / PingFang SC / Segoe UI） | 中英文字体优先级与桌面端一致 |

**桌面端设计 Token（移动端需映射）**：

## 四、不在范围内（Out of Scope）

以下功能**明确不做**：

| 功能 | 原因 |
|------|------|
| 富文档预览（PDF/DOCX/PPTX/XLSX） | 投入产出比低，可用系统自带应用打开 |
| 语音输入 | 与"轻交互"定位不符，IM 用户习惯打字 |
| 文件上传/下载 | 移动端不做文件管理 |
| 终端/浏览器面板 | 桌面专属能力 |
| 独立运行 Agent / Sidecar | 计算始终在桌面端 |
| Agent 创建/配置管理 | 在桌面端完成 |
| MCP 服务器管理 | 在桌面端完成 |

## 五、非功能需求

### 5.1 硬约束（必须达标才能发布）

| 指标 | 约束 | 说明 |
|------|------|------|
| Crash-free 率 | ≥ 99% | 基本稳定性 |
| SSE 流式延迟（局域网） | ≤ 200ms | 用户可感知的流畅度 |
| 配对认证安全 | JWT Token + 配对码 | 不能裸连 |
| 最低系统版本 | Android 8.0+ / iOS 15+ | 覆盖 95%+ 用户 |

### 5.2 目标（尽力而为，可降级）

| 指标 | 目标 | 降级阈值 |
|------|------|---------|
| 首屏加载时间 | ≤ 3s | > 5s 需优化 |
| 流式渲染帧率 | ≥ 25fps | < 15fps 需降级 |
| 内存占用 | ≤ 200MB | > 300MB 需优化 |
| Markdown 渲染一致性 | ≥ 85% | < 70% 需重写渲染器 |
| APK 体积 | ≤ 35MB | > 50MB 需裁剪 |
| 离线缓存大小 | ≤ 50MB | 超出时 LRU 淘汰 |

## 六、依赖与前提条件

| 依赖 | 说明 | 风险 |
|------|------|------|
| 桌面端 Sidecar 改造 | 需要支持局域网访问 + 配对码认证 + 移动端 API | 这是 MVP 的前置条件，需先合入桌面端主线 |
| 桌面端版本 | 需要包含 Sidecar 改造的版本发布 | 移动端发布前桌面端需先行发版 |
| 内网穿透 | 外网访问需要用户自行配置 Cloudflare Tunnel / frp | 不在移动端项目范围内，但需文档说明 |
